Contextualización del Diseño Arquitectónico

El diseño arquitectónico establece la estructura fundamental del sistema, conectando los requerimientos con la implementación. Nuestro sistema se compone de:

  • Componente de Reconocimiento Facial
  • Módulo de Notificaciones
  • Sistema de Reportes Automatizados
  • Interfaz Multiplataforma

Casos de Uso Principales

1. Registrar Asistencia

  • Actores: Cámara IA, Docente
  • Flujo Principal: Reconocimiento facial automático
  • Flujo Alternativo: Validación manual por docente

2. Generar Reportes

  • Formatos disponibles: PDF, Excel
  • Frecuencia: Diario, Semanal, Mensual
  • Acceso: Web y Móvil

Actores del Sistema

  • Cámara IA: Componente autónomo para reconocimiento facial
  • Administrador: Configuración del sistema y gestión de usuarios
  • Docente: Supervisión y validación de registros
  • Padres: Recepción de notificaciones en tiempo real

Proceso de Construcción

  1. Análisis de requerimientos con stakeholders
  2. Diseño de arquitectura modular
  3. Validación de casos de uso
  4. Prototipado de interfaces

Conclusión

El diseño arquitectónico presentado garantiza una base sólida para el desarrollo, cumpliendo con los objetivos de negocio y permitiendo una implementación escalable y mantenible.
