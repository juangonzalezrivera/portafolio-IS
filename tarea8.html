Componentes del Diseño Estructural

1. Diagrama de Bloques

Representación visual de alto nivel que muestra los componentes principales y sus interacciones.

  • Módulo de Reconocimiento Facial
  • Sistema de Notificaciones
  • Base de Datos Estudiantil
  • Interfaz de Reportes

2. Diseño Semántico de Datos

Modelado conceptual de entidades y sus relaciones:

Entidades:
- Estudiante (ID, Nombre, Grupo)
- RegistroAsistencia (Fecha, Hora, Estado)
- Clase (Materia, Horario)
Relaciones:
- Un Estudiante tiene múltiples RegistrosAsistencia
- Cada Clase contiene múltiples Estudiantes

3. Tarjeta CRC - Ejemplo

Clase: Estudiante

Responsabilidades:

  • Almacenar datos personales
  • Gestionar registros de asistencia
  • Notificar cambios de estado

Colaboradores:

  • RegistroAsistencia
  • Clase
  • SistemaNotificaciones

4. Diccionario de Clases

  • Estudiante:
    • Atributos: ID, nombre, grupo
    • Métodos: obtenerAsistencia(), actualizarDatos()
  • RegistroAsistencia:
    • Atributos: fecha, hora, estado
    • Métodos: generarReporte(), notificarAusencia()

5. Diagrama de Clases (UML)

                +---------------------+       +---------------------+
                |     Estudiante       |       | RegistroAsistencia  |
                +---------------------+       +---------------------+
                | - id: int           |<>-----| - fecha: Date       |
                | - nombre: String    |       | - hora: Time        |
                | - grupo: String     |       | - estado: boolean   |
                +---------------------+       +---------------------+
                | + getAsistencia()   |       | + generarReporte()  |
                | + actualizarDatos()|       +---------------------+
                +---------------------+

Implementación en el Sistema

  • Arquitectura modular basada en componentes
  • Integración con base de datos relacional
  • Patrón MVC para separación de responsabilidades
  • API REST para comunicación entre módulos
